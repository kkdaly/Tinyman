AI Agent 渐进式演进：三阶段架构

从极简原型到平台化的演进路线，每阶段都有明确的升级信号

阶段一：极简单 Agent（第 1-3 天）

Phase 1: 最小可行原型
┌──────────────┐     ┌──────────────┐     ┌──────────────────┐

                                          │  tmux 托管         │

核心要素：1 个 Agent + 1 份知识库 + prompt 驱动 + tmux 托管

升级信号 — 什么时候进入下一阶段？

  • 一→二：上下文开始混杂、多用户并发时回答质量下降、需要第二个角色分担
  • 二→三：用户量持续增长、单实例扛不住、需要工单追溯和数据分析

不要做的事

  • 不要一开始就搞 MQ / 数据库 / MCP 全套
  • 不要在没有验证场景价值前投入工程优化
  • 不要试图一次性做好所有 prompt
